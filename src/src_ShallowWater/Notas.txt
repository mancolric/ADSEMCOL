To achieve C-property, it is necessary that
	eta_h = h_h + b = const + O(1e-12)
If b is a function that cannot be represented in the finite element space, it is impossible that the latter is achieved. Therefore, we replace the latter condition by
	eta_h = h_h + b_h = const + O(1e-12)
with b_h the L2-projection of b. We also work with b_h and its derivatives in the code instead of b.

To achieve C-property, it is necessary that
	int( h_h nabla (h_h + b_h) ) = 0	
which is achieved if eta_h = h_h + b_h is constant. Equivalently, if q_h = 0 and eta_h = h_h + b_h = const, then it is possible to show:
1) the numerical fluxes are zero
2) the LIRK scheme yields constant solution. Note that y^i = y^1 + tau*sum_j a_ij^E f_j + tau*sum_j (a_ij^I-a_ij^E) * J*y^j and sum_j(a_ij^I-a_ij^E) = c_i - c_i = 0.

To ensure that b_h is the L2-projection of b at all time levels, even after mesh adaptation, we can set
db_h /dt = db_h/dt - db/dt = - tau_0*(b_h - b)
so that "errors" in b_h are quickly corrected. Indeed, the steady-state solution of the latter equation is int( (b_h-b) \psi ) = 0, that is, the L2-projection. "tau_0" should be a characteristic time, constant for all the domain maybe min((h/p)/lambda) so the CFL number appears in the discretization.

Fully well-balanced: satisface de forma "exacta" las soluciones estacionarias, es decir,
    div(F(u)) = 0
Un caso particular es el de "lake at rest", en el cual
    q1=q2=0 (se satisface div(F1)=0)
    gh*(dh/dx+db/dx)=0
    gh*(dh/dy+db/dy)=0
las dos últimas dan como solución eta=h+b=const. 

En la práctica, es imposible que la solución numérica proporcione la solución exacta, a menos que esta esté dentro del espacio de elementos finitos V_h. Por ejemplo, no se puede cumplir h_h + b = const salvo que b pertenezca a V_h.

Creo que sería mejor definir primero una "solución estacionaria" discretizada, es decir, una solución numérica ubar_h que satisfaga
    div(F(ubar_h)) = 0
Además, también hay que discretizar b, por ejemplo, b_h puede ser la proyección L2. De lo contrario, sigue sin poder cumplirse hbar_h+b=const en un caso general.

La ecuación anterior también se puede poner como
    int (div(F(ubar_h)) psi) = 0 \forall \psi
Lamentablemente, debido a la forma que tiene F, no es posible calcular esa integral de forma exacta, a no ser que se utilicen reglas de cuadratura especiales (para funciones racionales). Por tanto, si el esquema numérico alcanza un estado estacionario, se tendrá int(div(F(u_h) psi)=0, pero esto no implica div(F(u_h))=0, y que u_h=ubar_h, ya que la integral no se ha calculado exactamente.
